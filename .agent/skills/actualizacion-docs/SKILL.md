# Skill: Update Documentation

**Objetivo**: Mantener la documentación viva y sincronizada con la realidad del código.

## Instrucciones de Ejecución

1.  **Analizar Cambios**:
    - Revisar el historial reciente de cambios o los archivos modificados en la última tarea.
    - Identificar nuevas features, cambios en la arquitectura de carpetas, o modificaciones en modelos de datos (`types/`).

2.  **Verificar Discrepancias**:
    - Leer el archivo `BLUEPRINT.md` actual (o crear uno si no existe).
    - Comparar la estructura documentada vs. la estructura real del proyecto (`src/features`, etc.).

3.  **Acción de Actualización**:
    - Reescribir o enmendar las secciones desactualizadas del `BLUEPRINT.md`.
    - Si se crearon nuevas reglas o workflows, actualizarlos en sus respectivos archivos si es necesario.
    - Asegurar que el diagrama de arquitectura (si existe) refleje la realidad.

4.  **Confirmación**:
    - Reportar qué secciones fueron actualizadas y por qué.
