# Habilidad: Semantic Git & Version Control

## Objetivo
Mantener un historial de cambios limpio y profesional siguiendo la convención de "Semantic Commits".

## Formato de Commits
Usa siempre la estructura: `tipo(alcance): descripción breve`

* `feat`: Nueva funcionalidad.
* `fix`: Corrección de errores.
* `docs`: Cambios en documentación.
* `style`: Formato, puntos y comas (no lógica).
* `refactor`: Cambio de código que no arregla bugs ni añade features.
* `chore`: Tareas de mantenimiento, actualización de dependencias.

*Ejemplo:* `feat(auth): implementar login con Google y validación de tokens`
